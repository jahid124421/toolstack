# 📁 File Guide - What Each File Does

## 🚀 Quick Launch Files (BAT)

### `START-TEAM-ENHANCED.bat` ⭐ NEW
**The main launcher** - Use this instead of START-TEAM.bat
- Interactive menu for mode selection
- Choose CODING, BA, or AUTO mode
- Shows provider priority and token limits
- Saves your mode preference

### `START-TEAM.bat`
Original launcher (still works)
- Uses AUTO mode only
- No mode selection
- Keep for backward compatibility

### `SHOW-USAGE.bat` ⭐ NEW
Quick usage stats viewer
- Shows current provider and model
- Displays token usage
- One-time snapshot
- Fast and simple

### `COMPARE-MODES.bat` ⭐ NEW
Interactive mode comparison tool
- Compare CODING vs BA modes
- View provider priorities
- Check token limits
- Educational tool

### `deploy_now.bat`
Deployment script (parent directory)
- Not related to AI agent
- For deploying your main project

## 🐍 Python Core Files

### `ai-agent-team.py` ⭐ ENHANCED
**The main server** - Enhanced with usage tracking
- Multi-agent AI team orchestration
- OpenAI-compatible API endpoint
- Real-time usage tracking
- Provider fallback logic
- Role-based model caching
- Token counting and logging

**New Features:**
- `_USAGE_STATS` global tracking
- `estimate_tokens()` function
- Usage data in all responses
- `/usage` endpoint
- Enhanced `/` status endpoint

### `show_usage.py` ⭐ NEW
Usage monitoring script
- Real-time usage display
- Continuous monitoring mode (`--watch`)
- Formatted output with colors
- Provider and model info
- Token breakdown
- Cost tracking for paid APIs

### `fetch_models.py`
Fetch available models from providers
- Queries GROQ, CEREBRAS, CF, OR, FM
- Updates `available_models.json`
- Useful for discovering new models

### `select_model.py`
Command-line model selector
- Legacy tool for model selection
- Reads from `available_models.json`
- Used by older BAT files

### `ADMIN-SERVER.py`
Web-based admin UI (separate server)
- Provider management interface
- Model tier configuration
- Runs on same port as main server
- Access: `http://localhost:8080/admin`

### `test_passthrough.py`
Test script for passthrough mode
- Used for development/debugging
- Not needed for normal use

## 📄 Configuration Files

### `my-keys.env` 🔒
**API keys storage** (keep private!)
```env
GROQ_API_KEY=gsk_...
CEREBRAS_API_KEY=csk_...
CLOUDFLARE_ACCOUNT_ID=...
CLOUDFLARE_API_TOKEN=cfut_...
FREEMODEL_KEY=fe_oa_...
OPENROUTER_API_KEY=sk-or-...
ANTHROPIC_API_KEY=sk-ant-... (optional, paid)
```

### `available_models.json`
List of models from each provider
- Auto-generated by `fetch_models.py`
- Used for model selection
- Updates periodically

### `current_model.txt` ⭐ ENHANCED
Currently selected model
- Written by enhanced launcher
- Read by server on startup
- Can be edited manually

### `selected_model.txt`
Legacy model selection file
- Used by older scripts
- May be deprecated

### `mode_config.txt` ⭐ NEW
Saved mode preference
- Written by `START-TEAM-ENHANCED.bat`
- Options: CODING, BA_WORK, AUTO
- Persists between sessions

### `admin_config.json`
Admin UI configuration
- Generated by ADMIN-SERVER.py
- Provider settings
- Model tier mappings

### `passthrough.on`
Passthrough mode flag
- Enables passthrough functionality
- Used in testing

## 🔍 Finding Files

### Windows Command Prompt
```bat
cd ai-bots-package
dir *.bat         # List all BAT files
dir *.py          # List all Python files
dir *.md          # List all documentation
type FILE.md      # Read a file
```

### View Usage Stats
```bat
SHOW-USAGE.bat
```

### View Logs
```bat
type ai-agent-team.log
type ai-agent-team.log | findstr "usage"  # Filter for usage lines
type ai-agent-team.log | findstr "error"  # Filter for errors
```

### Check Current Mode
```bat
type mode_config.txt
type current_model.txt
```
