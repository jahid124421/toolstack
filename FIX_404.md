# Fix 404 Error - Domain Works But Site Not Loading

## Problem
- Domain `toolstack.dpdns.org` is working (DNS resolved)
- But you're seeing 404 error
- This means: Domain is connected, but site isn't deployed or custom domain not set up

## Solution (2 Steps)

### Step 1: Deploy the Site (REQUIRED)

The site files need to be deployed to Cloudflare Pages first.

**Method A: Cloudflare Dashboard (Easiest)**
1. Go to: https://dash.cloudflare.com/pages
2. Click your **toolstack** project
3. You should see "No deployments yet" or old deployments
4. Click **Deployments** → **Create deployment**
5. Select **Upload assets** (or drag & drop)
6. Select the entire `ToolStack` folder
7. Click **Deploy**
8. Wait 2-3 minutes

**Method B: Wrangler CLI**
```bash
cd ToolStack
wrangler login
wrangler pages deploy
```

### Step 2: Add Custom Domain (REQUIRED)

After deployment, you MUST connect the custom domain:

1. Go to: https://dash.cloudflare.com/pages
2. Click your **toolstack** project
3. Go to **Custom domains** tab (left sidebar)
4. Click **Set up a custom domain**
5. Enter: `toolstack.dpdns.org`
6. Click **Continue**
7. Click **Activate domain**

Cloudflare will automatically:
- Create DNS record
- Issue SSL certificate
- Enable HTTPS

### Step 3: Wait and Test

1. Wait 2-5 minutes for DNS propagation
2. Visit: https://toolstack.dpdns.org
3. The site should load now!

## Troubleshooting

### Still Getting 404?

**Check 1: Is the site deployed?**
- Go to: https://dash.cloudflare.com/pages
- Click **toolstack** project
- Check **Deployments** tab
- You should see at least 1 successful deployment
- If not, deploy again (Step 1 above)

**Check 2: Is custom domain added?**
- Go to: https://dash.cloudflare.com/pages
- Click **toolstack** project
- Go to **Custom domains** tab
- You should see `toolstack.dpdns.org` listed
- If not, add it (Step 2 above)

**Check 3: Is domain active?**
- In Custom domains tab, check status
- Should say "Active" (not "Pending" or "Error")
- If pending, wait 5-10 more minutes

**Check 4: Clear cache**
- Clear browser cache (Ctrl+Shift+R)
- Try incognito mode
- Try different browser

### Still Not Working?

**Verify deployment files:**
```bash
cd ToolStack
dir
```
You should see:
- index.html
- sitemap.xml
- css/ folder
- js/ folder
- tools/ folder

**Check deployment logs:**
- Go to Cloudflare Pages dashboard
- Click on latest deployment
- Check build logs for errors

**Try redeploying:**
1. Delete current deployment
2. Create new deployment
3. Upload files again

## Common Mistakes

❌ **Mistake 1:** Added domain to Cloudflare DNS but didn't deploy site
- **Fix:** Deploy site first, then add custom domain

❌ **Mistake 2:** Added custom domain but site not deployed
- **Fix:** Deploy site first

❌ **Mistake 3:** Deployed but didn't wait for DNS propagation
- **Fix:** Wait 5-10 minutes

❌ **Mistake 4:** Deployed wrong folder
- **Fix:** Make sure you're deploying the `ToolStack` folder contents
